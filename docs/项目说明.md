# 项目说明

## 1. 项目定位

EPMIS 用于支撑环保与环境管理相关业务，当前代码仓库主要包含认证、平台业务、数据处理、协议接入四类后端服务。整体架构以模块拆分方式组织，便于服务独立运行、独立维护和接口联调。

## 2. 模块说明

### 2.1 聚合与公共模块

- `pom.xml`：顶层聚合工程，统一管理各子模块
- `epmis-parent`：统一依赖版本与编译配置
- `epmis-core`：公共常量、工具类、响应对象、Redis 封装、分页对象等
- `epmis-generator`：代码生成辅助模块

### 2.2 业务服务模块

#### 认证服务 `epmis-auth`

职责：

- 用户登录认证
- Token 签发与用户上下文获取
- 用户、角色、岗位、菜单、配置、公告管理
- 缓存、在线用户、日志监控、文件上传下载

关键接口分组：

- `/login`
- `/getInfo`
- `/getRouters`
- `/system/user/**`
- `/system/role/**`
- `/system/menu/**`
- `/system/config/**`
- `/monitor/**`
- `/common/**`

#### 平台服务 `epmis-platform`

职责：

- 企业主数据与基础字典管理
- 环保业务台账管理
- 应急管理模块
- 审批、附件、PLC、运维、首页、AI 等业务支持
- 对其他服务提供部分 Feign 内部接口

主要接口分组：

- `/platform/dict/**`
- `/platform/ent/**`
- `/platform/emergency/**`
- `/platform/envProtect/**`
- `/platform/envFee/**`
- `/platform/approval/**`
- `/platform/ops/**`
- `/platform/plc/**`
- `/platform/feign/**`

#### 数据处理服务 `epmis-process`

职责：

- 在线监测数据统计分析
- 固废业务处理
- 有效传输率与企业排放统计
- 报警处理、执法记录、首页统计
- SSE 推送与异步导出

主要接口分组：

- `/process/onlineMonitoring/**`
- `/process/wasteCategory/**`
- `/process/wasteRoom/**`
- `/process/wasteStock/**`
- `/process/wasteStat/**`
- `/process/validPeriod/**`
- `/process/emission/**`
- `/process/durAlarm/**`
- `/process/homePage/**`
- `/export/**`

#### 协议服务 `epmis-protocol`

职责：

- 协议指令发送
- 设备在线连接监控
- Netty、Modbus TCP、MQTT 等接入能力

主要接口分组：

- `/protocol/cmd/**`
- `/protocol/connections/**`

## 3. 技术方案

### 3.1 基础框架

- 基于 `Spring Boot 3.2.5`
- 使用 `Spring Cloud 2023.0.1`
- 使用 `MyBatis` 访问数据库
- 使用 `Redis` 处理缓存、Token 或业务辅助能力
- 使用 `JWT` 进行令牌解析与身份识别

### 3.2 代码组织方式

各业务一般采用如下结构：

- `controller`：接口入口
- `service` / `service.impl`：业务实现
- `domain`：实体、请求对象、响应对象
- `mapper`：数据库访问层
- `resources/mapper/*.xml`：SQL 映射文件

### 3.3 数据交换方式

- 认证服务负责签发 Token
- 平台、流程等服务通过请求头传递 `Authorization`
- 服务间通过 Feign 进行 HTTP 调用
- 部分业务通过文件模板完成 Excel 导入导出

## 4. 服务端口与依赖关系

| 服务 | 模块 | 默认端口 | 主要依赖 |
| --- | --- | --- | --- |
| 认证服务 | `epmis-auth-service` | `9531` | MySQL、Redis |
| 数据处理服务 | `epmis-process-service` | `9532` | MySQL、Redis、认证服务、平台服务 |
| 平台服务 | `epmis-platform-service` | `9533` | MySQL、Redis、认证服务、协议服务 |
| 协议服务 | `epmis-protocol-service` | `9534` | 平台服务、Netty、MQTT、Modbus |

## 5. 配置说明

各服务均使用 `src/main/resources/application.yml` 管理配置，重点包括：

- `server.port`：服务端口
- `spring.datasource`：MySQL 连接
- `spring.data.redis`：Redis 配置
- `feign.*.url`：服务间调用地址
- `logging.file.name`：日志输出位置
- `token.*`：认证相关参数
- `mqtt.*`、`modbus.*`、`xxl.job.*`：扩展组件配置

建议：

- 开发、测试、生产环境使用独立配置文件或外部配置注入
- 账号、密码、密钥等敏感参数通过环境变量或配置中心管理

## 6. 资源与模板

项目中存在大量导入导出模板，主要位于：

- `epmis-platform/epmis-platform-service/src/main/resources/template/`
- `epmis-process/epmis-process-service/src/main/resources/template/`

这些模板覆盖企业信息、应急模块、环保费用、固废、在线监测等场景，是前后端联调和业务导入的重要配套资源。

## 7. MQTT 消息链路

项目中的 MQTT 主要承担协议层到数据处理层的异步消息转发，不直接作为 HTTP 接口暴露。

链路角色：

- `epmis-protocol-service`：发布端
- `epmis-process-service`：订阅端

主要 topic：

- `out_put_data_real`
- `out_put_data_minute`
- `out_put_data_hour`
- `out_put_data_day`
- `device_data_plc`

典型链路：

1. 设备通过 `HJ212` 或 `Modbus TCP` 接入 `epmis-protocol-service`
2. 协议服务解析报文或寄存器数据后发布到 MQTT Broker
3. `epmis-process-service` 订阅对应 topic 并按类型分流处理
4. 数据处理服务完成监测数据落库、PLC 数据保存、缓存更新和报警计算

典型用途：

- 协议服务将 HJ212 解析结果发布到 MQTT
- 协议服务将 PLC 原始采集数据发布到 MQTT
- 数据处理服务订阅消息后进行落库、缓存更新和报警处理

关键配置项：

- 协议服务：`mqtt.broker-url`、`mqtt.client-id`、`mqtt.username`、`mqtt.password`
- 数据处理服务：`mqtt.enabled`、`mqtt.broker-url`、`mqtt.client-id`、`mqtt.username`、`mqtt.password`、`mqtt.qos`

使用建议：

- 发布端和订阅端应接入同一个 Broker
- 不同环境使用不同 `client-id`
- 生产环境通过环境变量或配置中心管理 MQTT 账号密码
