# 数据库初始化说明

## 1. 说明

本项目仓库已提供数据库建表与初始化数据脚本，但 `sql/` 目录中的脚本不包含 `CREATE DATABASE` 或 `USE` 语句，需要先手动创建目标数据库，再在对应数据库中执行脚本。

## 2. 当前配置现状

根据当前仓库中的示例配置：

- `epmis-auth-service` 使用数据库：`zkhf_poll_test`
- `epmis-platform-service` 使用数据库：`zkhf_poll_test`
- `epmis-process-service` 使用数据库：`zkhf_pollution`

建议：

- 本地联调前先统一三个服务的 MySQL 数据源地址和库名
- 若希望所有服务共用一套数据，建议统一到同一个库
- 若按现有配置拆分库运行，则需要确认业务表和基础表是否都已同步到对应库

## 3. SQL 目录说明

`sql/` 目录结构如下：

- `sql/zkhf_pollution_basic.sql`：系统基础表、权限表、字典表、平台基础业务表
- `sql/zkhf_pollution_data.sql`：在线监测、报警、有效期、固废、PLC 等业务数据表
- `sql/emergency_tables.sql`：应急管理扩展表
- `sql/material_management_tables.sql`：物资管理业务表（台账、仓库、库存、入库/出库/归还/申请及明细、库存流水）
- `sql/data/*.sql`：初始化数据脚本，包括用户、菜单、字典、配置、基础字典等

## 4. 推荐初始化顺序

推荐按以下顺序执行：

1. 手动创建数据库
2. 执行 `sql/zkhf_pollution_basic.sql`
3. 执行 `sql/zkhf_pollution_data.sql`
4. 执行 `sql/emergency_tables.sql`
5. 执行 `sql/material_management_tables.sql`
6. 依次执行 `sql/data/` 目录中的初始化数据脚本

推荐的数据脚本执行顺序：

1. `sql/data/sys_config.sql`
2. `sql/data/sys_dict_type.sql`
3. `sql/data/sys_dict_data.sql`
4. `sql/data/sys_post.sql`
5. `sql/data/sys_user.sql`
6. `sql/data/sys_user_post.sql`
7. `sql/data/sys_user_role.sql`
8. `sql/data/sys_menu.sql`
9. `sql/data/t_bas_districts.sql`
10. `sql/data/t_bas_industry_category.sql`
11. `sql/data/t_bas_pollutant_code.sql`
12. `sql/data/t_bas_fee_type_dict.sql`
13. `sql/data/t_bas_fee_status_dict.sql`
14. `sql/data/t_bas_payment_method_dict.sql`
15. `sql/data/t_bas_payment_status_dict.sql`
16. `sql/data/t_valid_period_conf.sql`
17. `sql/data/t_waste_dict.sql`

说明：

- `sys_user_post.sql` 依赖 `sys_user.sql` 和 `sys_post.sql`
- `sys_user_role.sql` 依赖 `sys_user` 与角色基础数据
- 若重复执行初始化数据，可能触发主键冲突

## 5. 建库示例

如果你准备统一使用单库联调，可先手动创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS zkhf_pollution
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_0900_ai_ci;
```

或统一为：

```sql
CREATE DATABASE IF NOT EXISTS zkhf_poll_test
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_0900_ai_ci;
```

创建后，在数据库客户端中切换到目标库，再执行脚本文件。

## 7. 初始化数据说明

当前仓库已提供一批基础初始化数据，包括：

- 系统参数 `sys_config`
- 字典类型与字典数据
- 初始岗位
- 初始用户
- 用户岗位关系
- 用户角色关系
- 初始菜单
- 行政区划、行业分类、污染物编码
- 支付方式、费用状态、固废字典、有效期配置

其中：

- `sys_config.sql` 中包含 `sys.user.initPassword=123456`
- `sys_config.sql` 中包含 `sys.account.captchaEnabled=true`
- `sys_user.sql` 中已插入管理员账号 `admin`

注意：

- `sys.user.initPassword` 是新建用户的初始化密码配置，不等同于当前数据库中 `admin` 账号一定可直接使用该明文密码
- 实际登录密码以 `sys_user.password` 中保存的 BCrypt 密文为准

## 8. 初始化后核查项

建议执行以下检查：

1. 检查系统用户表是否已有 `admin`
2. 检查 `sys_menu` 是否已导入菜单
3. 检查 `sys_config` 是否存在验证码和初始化密码配置
4. 检查应急相关表是否已创建
5. 检查 `t_bas_*` 字典基础数据是否已导入

可参考以下 SQL：

```sql
SELECT user_id, user_name, nick_name, status FROM sys_user;
SELECT config_key, config_value FROM sys_config
WHERE config_key IN ('sys.user.initPassword', 'sys.account.captchaEnabled');
SHOW TABLES LIKE 't_emergency_%';
```

## 9. 常见问题

### 9.1 执行脚本后服务仍报表不存在

排查方向：

- 是否导入到了错误的数据库
- 服务 `application.yml` 的数据库名是否和导入目标一致
- 是否漏执行了 `zkhf_pollution_data.sql` 或 `emergency_tables.sql`

### 9.2 菜单或字典为空

排查方向：

- 是否执行了 `sql/data/sys_menu.sql`
- 是否执行了 `sql/data/sys_dict_type.sql` 和 `sql/data/sys_dict_data.sql`

### 9.3 登录失败

排查方向：

- `sys_user` 是否存在 `admin`
- `sys.account.captchaEnabled` 为 `true` 时是否先调用了验证码接口
- Redis 是否正常
- 实际账号密码是否与数据库中的密文匹配

## 10. 建议

- 开发环境优先统一使用单库，减少跨服务联调复杂度
- 首次初始化完成后，保留一份可重复执行的数据库快照
- 生产环境请拆分账号权限，不要复用仓库中的示例配置
