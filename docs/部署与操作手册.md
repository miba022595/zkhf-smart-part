# 部署与操作手册

## 1. 适用范围

本文档适用于本地开发、测试环境部署、接口联调以及基础运维排查。

## 2. 环境要求

### 2.1 软件依赖

- `JDK 21`
- `Maven 3.9+`
- `MySQL 8.x`
- `Redis 6.x/7.x`

可选组件：

- `MQTT Broker`：协议层或消息订阅相关场景
- `XXL-JOB`：定时任务相关场景
- `企业微信配置`：消息通知相关场景
- `Modbus TCP` / `Netty`：协议接入场景

### 2.2 机器建议

- CPU：`4 Core+`
- 内存：`8 GB+`
- 磁盘：预留日志与导出文件空间

## 3. 关键目录

- 根目录：聚合工程入口
- `epmis-auth/epmis-auth-service`：认证服务启动模块
- `epmis-platform/epmis-platform-service`：平台服务启动模块
- `epmis-process/epmis-process-service`：流程处理服务启动模块
- `epmis-protocol/epmis-protocol-service`：协议服务启动模块
- `api/`：接口文档
- `docs/`：说明文档与手册

## 4. 部署前准备

### 4.1 修改配置文件

请根据目标环境修改以下文件：

- `epmis-auth/epmis-auth-service/src/main/resources/application.yml`
- `epmis-platform/epmis-platform-service/src/main/resources/application.yml`
- `epmis-process/epmis-process-service/src/main/resources/application.yml`
- `epmis-protocol/epmis-protocol-service/src/main/resources/application.yml`

重点确认以下配置：

- `server.port`
- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`
- `spring.data.redis.host`
- `spring.data.redis.port`
- `feign.*.url`
- `logging.file.name`
- `token.secret`
- `mqtt.*`
- `modbus.*`
- `xxl.job.*`

### 4.2 检查数据库与缓存

- 确认 MySQL 服务可访问
- 确认 Redis 服务可访问
- 确认数据库账号具备读写权限

## 5. 本地启动方式

### 5.1 编译

在仓库根目录执行：

```bash
mvn clean install -DskipTests
```

### 5.2 单服务启动命令

认证服务：

```bash
mvn -pl epmis-auth/epmis-auth-service -am spring-boot:run
```

平台服务：

```bash
mvn -pl epmis-platform/epmis-platform-service -am spring-boot:run
```

数据处理服务：

```bash
mvn -pl epmis-process/epmis-process-service -am spring-boot:run
```

协议服务：

```bash
mvn -pl epmis-protocol/epmis-protocol-service -am spring-boot:run
```

### 5.3 推荐启动顺序

1. `epmis-auth-service`
2. `epmis-platform-service`
3. `epmis-process-service`
4. `epmis-protocol-service`

说明：

- `process` 依赖认证服务和平台服务地址
- `platform` 依赖认证服务和协议服务地址
- `protocol` 依赖平台服务地址

## 6. 打包部署方式

### 6.1 打包

在根目录执行：

```bash
mvn clean package -DskipTests
```

生成的可执行 Jar 位于各服务模块 `target/` 目录。

### 6.2 Jar 启动

认证服务：

```bash
java -jar epmis-auth/epmis-auth-service/target/epmis-auth-service.jar
```

平台服务：

```bash
java -jar epmis-platform/epmis-platform-service/target/epmis-platform-service.jar
```

数据处理服务：

```bash
java -jar epmis-process/epmis-process-service/target/epmis-process-service.jar
```

协议服务：

```bash
java -jar epmis-protocol/epmis-protocol-service/target/epmis-protocol-service.jar
```

### 6.3 外部化配置示例

```bash
java -jar epmis-platform-service.jar --spring.config.location=file:/data/epmis/config/application.yml
```

## 8. 常用操作

### 8.1 获取 Token

请求：

```http
POST /login
Content-Type: application/json

{
  "username": "admin",
  "password": "******",
  "code": "1234",
  "uuid": "captcha-uuid"
}
```

登录成功后，在后续请求头中携带：

```http
Authorization: Bearer <token>
```

### 8.2 上传文件

常用上传入口：

- `POST /common/upload`
- `POST /common/uploads`

### 8.3 下载导入模板

项目大量业务提供 Excel 模板下载与导入导出能力，模板文件主要位于：

- `epmis-platform-service/src/main/resources/template/`
- `epmis-process-service/src/main/resources/template/`

### 8.4 Windows 联调脚本

仓库内已提供应急模块联调脚本：

- `test/emergency_curl_test_windows.cmd`

适用于在 Windows 环境中快速验证应急接口。

## 9. 日志与排查

默认日志文件：

- 认证服务：`logs/auth-app.log`
- 平台服务：`logs/platform-app.log`
- 数据处理服务：`logs/process-app.log`
- 协议服务：`logs/protocol-app.log`

排查建议：

1. 先确认端口是否监听成功
2. 再检查数据库与 Redis 连通性
3. 再检查 Feign 依赖服务是否可访问
4. 最后检查业务接口入参与权限

## 10. 常见问题

### 10.1 登录失败

排查方向：

- 验证码是否正确
- Redis 是否正常
- 用户是否被锁定
- `token.secret` 配置是否一致

### 10.2 业务接口 401/403

排查方向：

- 是否携带 `Authorization`
- Token 是否过期
- 当前用户是否具备菜单与数据权限

### 10.3 Feign 调用失败

排查方向：

- `feign.*.url` 是否配置正确
- 被依赖服务是否启动
- 网络访问或防火墙是否拦截

### 10.4 Excel 导入失败

排查方向：

- 是否使用系统提供的模板
- 表头、日期、枚举值格式是否符合要求
- 导入文件大小是否超过限制

### 10.5 MQTT 相关问题

排查方向：

- MQTT Broker 是否已启动
- `mqtt.broker-url` 是否正确
- 发布端与订阅端是否使用同一 Broker
- `client-id` 是否冲突
- `epmis-process-service` 是否开启 `mqtt.enabled`
- topic 名称是否一致

## 11. MQTT 部署与验证

### 11.1 涉及服务

- `epmis-protocol-service`：MQTT 发布端
- `epmis-process-service`：MQTT 订阅端

链路说明：

1. 设备通过 `HJ212` 或 `Modbus TCP` 接入协议服务
2. 协议服务将解析后的监测数据或 PLC 原始数据发布到 MQTT
3. 数据处理服务订阅 topic 后进行落库、缓存更新和报警处理

### 11.2 关键配置

协议服务需要配置：

- `mqtt.broker-url`
- `mqtt.client-id`
- `mqtt.username`
- `mqtt.password`

数据处理服务需要配置：

- `mqtt.enabled`
- `mqtt.broker-url`
- `mqtt.client-id`
- `mqtt.username`
- `mqtt.password`
- `mqtt.qos`

建议配置示例：

```yaml
mqtt:
  enabled: true
  broker-url: tcp://<broker-host>:1883
  client-id: epmis-process-subscriber
  username: <username>
  password: <password>
  qos: 1
  connection-timeout: 30
  keep-alive-interval: 60
  clean-session: true
```

配置说明：

- `broker-url`：MQTT Broker 地址
- `client-id`：客户端唯一标识，不能重复
- `enabled`：是否启用数据处理服务的 MQTT 订阅
- `qos`：订阅质量等级
- `connection-timeout`：连接超时时间
- `keep-alive-interval`：心跳保活时间
- `clean-session`：是否使用清理会话模式

### 11.3 推荐启用顺序

1. 启动 MQTT Broker
2. 启动 `epmis-process-service`
3. 启动 `epmis-protocol-service`
4. 接入 HJ212 或 Modbus 设备

### 11.4 核心 topic

- `out_put_data_real`
- `out_put_data_minute`
- `out_put_data_hour`
- `out_put_data_day`
- `device_data_plc`

topic 含义：

- `out_put_data_real`：在线监测实时数据
- `out_put_data_minute`：在线监测分钟数据
- `out_put_data_hour`：在线监测小时数据
- `out_put_data_day`：在线监测日数据
- `device_data_plc`：PLC 原始采集数据

### 11.5 验证方式

- 查看 Broker 客户端连接状态
- 查看协议服务日志是否出现 MQTT 发送记录
- 查看数据处理服务日志是否出现 MQTT 接收记录
- 检查在线监测数据、PLC 数据是否已落库
- 检查前端实时页面或 Redis 缓存是否出现最新数据

## 12. 生产部署建议

- 使用外部配置文件管理环境差异
- 使用独立账号管理数据库、Redis、MQTT 等连接
- 将密钥、密码、令牌配置迁移到安全存储
- 配置日志切割与归档目录
- 对外开放端口前配置网关、鉴权与防火墙策略
