# 物资管理需求说明

## 1. 模块概述

物资管理用于对企业日常物资进行统一建档、库存跟踪、入库出库、领用申请和归还闭环管理，实现物资从基础信息维护到库存变动全过程留痕。

模块包含以下功能：

1. 物资查询
2. 库存管理
3. 物资入库
4. 物资申请
5. 物资出库
6. 物资归还

## 2. 总体目标

1. 建立统一的物资基础台账，支持按企业维度进行权限隔离。
2. 建立库存台账，实时反映物资可用库存、在库库存及出入库记录。
3. 支持批量导入、模板下载、导出等常用数据操作。
4. 支持申请、审核、出库、归还等完整业务流转。
5. 所有关键操作需保留操作日志，便于审计追溯。

## 3. 通用业务规则

### 3.1 企业权限控制

1. 所有列表查询、详情查看、新增、编辑、导入、导出、审核等操作均需进行企业权限控制。
2. 平台管理员可查看全部企业数据。
3. 企业用户仅可查看和操作所属权限范围内企业的数据。
4. 涉及“可用物资”的场景，物资选择范围仅展示当前用户有权限企业下的可用物资数据。
5. 单据关联企业后，后续流转环节默认继承该企业权限，不允许跨企业串用物资。

### 3.2 编码与唯一性规则

1. 物资编号应支持手工录入或系统生成，编号在同一企业范围内唯一。
2. 入库单号、出库单号、归还单号、申请单号应系统自动生成，支持按日期或业务前缀生成。
3. 到货单号支持手工录入，用于关联采购或到货业务。

### 3.3 库存计算规则

1. 入库单审批通过后增加库存。
2. 审核通过并完成出库后扣减库存。
3. 归还单审批通过并确认入库后增加库存。
4. 草稿、已保存、待审核、已驳回、已作废/已取消的业务单据不计入有效库存。
5. 出库单“已保存”或“提交审批”时占用库存：冻结数量增加、可用库存减少；审批通过后从冻结库存扣减在库库存；取消审批/驳回/作废时释放冻结数量。
6. 库存按“企业 + 仓库 + 物资”维度进行统计。

### 3.4 数据留痕要求

1. 所有新增、编辑、删除、导入、导出、审核、出库、归还等操作需记录操作人、操作时间、操作内容。
2. 关键单据需保留审核状态、审核人、审核时间、审核意见。
3. 重要业务单据建议支持附件上传，如申请附件、签收附件、归还说明附件等。

### 3.5 审批集成（平台统一审批 + MQTT 回调）

1. 物资模块不再由各业务接口直接完成“审核通过/驳回”的最终裁决；审核动作统一在平台层审批中心完成。
2. 发起审批统一调用平台 `ApprovalInstanceController.add` 创建审批实例（`t_bas_approval_instance`），业务数据以 `business_type + business_key` 关联。
3. 物资业务侧不提供单独的“提交审批”接口；前端点击“提交审批”时按以下顺序执行：
   - 调用业务侧“编辑”接口，将单据状态变更为“待审核/待审批”（状态=2）
   - 调用平台 `ApprovalInstanceController.add` 创建审批实例
4. 平台创建审批实例成功后，通过 MQTT 发布 `PROCESSING` 回调；业务层订阅并落库更新业务单据状态（用于兜底与最终一致）。
5. 平台审批到达终态（通过/驳回/取消）后，通过 MQTT 发布终态结果，业务层订阅并落库更新业务状态与库存。
6. MQTT 主题：`approval_result`。
7. 消息体字段（JSON）：
   - `businessType`：`material_apply` / `material_in` / `material_out` / `material_return`
   - `businessKey`：业务单据主键（如 `applyId`/`inId`/`outId`/`returnId`）
   - `entCode`：企业编码
   - `status`：`PROCESSING` / `APPROVED` / `REJECTED` / `CANCELLED`
   - `comment`：审批意见
   - `actionUserId`：审批人ID（可选）
   - `actionUserName`：审批人姓名（可选）
   - `actionTime`：操作时间（可选，格式：yyyy-MM-dd HH:mm:ss）
8. 幂等要求：同一业务单据重复收到终态消息时，不允许重复扣减/回补库存。

#### 4.1.6 导入要求

1. 支持下载标准导入模板。
2. 支持 Excel 批量导入。
3. 导入字段至少包含：企业、物资编号、物资名称、品牌、型号/规格型号、分类、单位、单价、初始库存。
4. 导入时需校验必填项、编号唯一性、数值格式合法性。
5. 导入结果需反馈成功条数、失败条数及失败原因。

#### 4.1.7 业务规则

1. 物资名称、分类、单位为基础必填项。
2. 单价、库存应为非负数。
3. 已发生业务流转的物资原则上不允许物理删除，可改为停用。
4. 停用物资不允许继续发起新的申请、出库和入库业务选择。

#### 4.1.8 权限控制

1. 查询范围按企业权限控制。
2. 新增、编辑、删除、导入、导出均需校验所属企业权限。

#### 4.2.5 库存明细

库存明细应展示具体台账流水，至少包含：

1. 业务类型（入库、出库、归还、调整）
2. 业务单号
3. 变动数量
4. 变动前库存
5. 变动后库存
6. 操作人
7. 操作时间
8. 备注

#### 4.2.6 业务规则

1. 当前库存由有效入库、出库、归还业务自动计算，不允许人工直接修改。
2. 可用库存 = 当前库存 - 已占用/待出库数量。
3. 当当前库存小于等于最低库存预警值时，标记为低库存预警。

#### 4.2.7 权限控制

1. 查询范围按企业权限控制。
2. 可查看物资范围仅限当前用户有权限企业下的可用物资。

3. 审核人仅可审核有权限企业范围内的申请单。
4. 申请单中可选物资仅展示用户有权限企业下的可用物资。

1. 出库单可来源于申请单，也可支持直接出库。

3. 审核通过并确认出库后扣减库存。
4. 实际出库数量必须大于 0，且不得超过可用库存。
5. 若存在审核流程，只有审核通过状态才允许库存生效。
6. 出库完成后应回写申请单出库状态，如未出完、已出完。

1. 查询和维护按企业权限控制。
2. 仓库、物资、申请单选择范围仅限有权限企业数据。

5. 已取消
6. 已保存

### 5.3 入库/出库/归还单状态

0. 已作废
1. 草稿
2. 待审核
3. 已审核
4. 已完成
6. 已保存

## 6. 导出要求

1. 物资查询支持导出基础台账信息。
2. 库存管理支持导出库存汇总及库存明细。
3. 物资申请支持导出申请单及申请明细。
4. 入库、出库、归还均支持按筛选条件导出对应单据数据。
5. 导出结果需遵循企业权限控制。

## 7. 非功能要求

1. 列表页支持分页查询。
2. 查询条件支持组合筛选。
3. 金额字段统一保留两位小数。
4. 数量字段支持按业务需要保留相应精度。
5. 页面操作响应应清晰提示成功、失败及原因。
6. 批量导入和导出应考虑大数据量场景下的性能和超时处理。
