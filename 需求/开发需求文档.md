## 一、项目概述

### 1.1 平台定位

面向园区管委会或园区生态环境管理分局，建设"监测—预警—溯源—处置—归档"一体化智慧管控中枢平台。实现"看得见污染、说得清来源、管得住排放、落得实责任"。

### 1.2 核心业务能力

| 能力 | 说明 |
|------|------|
| 立体感知 | 整合无人机、高空鹰眼、在线监测、视频监控，构建"点线面、天空地"一体化感知网络 |
| 智能预警 | 多级阈值+突变规则+AI预测，自动识别异常排放 |
| 闭环处置 | 预警→任务→派发→处置→反馈→归档→AI核实，全流程线上闭环 |
| 精准溯源 | 气象+排放特征+GIS+AI大模型，快速锁定水/气污染源头 |
| 政企协同 | 监管端与企业端打通，任务下发/整改反馈/数据上报线上协同 |

### 1.3 建设范围

- **智能感知网络**：废气/废水/空气/水质/视频/噪声/恶臭/VOCs/用电量/治理设施工况等监测设备接入
- **管控平台软件**：9大子系统（数据中心、一张图、智能监控、分级预警、闭环管理、溯源分析、档案管理、移动端、企业端）
- **部署环境**：阿里云

### 1.4 技术栈

| 类别 | 技术 |

### 1.5 现有系统基础

当前已有以下服务模块可复用：
- `epmis-auth`：认证服务（登录、Token、用户/角色/菜单管理）
- `epmis-platform`：平台服务（企业主数据、字典、应急、审批、运维等）
- `epmis-process`：数据处理服务（在线监测统计、固废、报警、导出等）
- `epmis-protocol`：协议服务（HJ212、Modbus TCP、MQTT接入）
- `epmis-core`：公共模块（工具类、响应对象、Redis封装、分页）

### 2.2 权限模型

```
权限维度：
├── 功能权限：菜单 + 按钮级权限（基于现有RBAC体系扩展）
├── 数据权限：
│   ├── 园区级：可查看所有企业数据
│   ├── 企业级：仅可查看本企业数据（企业端隔离）
│   ├── 环保分局级：可查看管辖片区数据
│   └── 第三方级：仅可查看被授权企业数据
└── 操作权限：
    ├── 查看：所有角色默认可查看授权范围内数据
    ├── 处置：环保管理人员、企业环保人员
    ├── 派发：环保管理人员、执法人员
    ├── 审核：安环部负责人、局执法负责人
    └── 管理：系统管理员
```

### 2.3 核心业务流程

**流程一：自动发现异常**
```
无人机/视频/在线监测/AI模型 → 发现数据异常
  → 系统自动预警
  → 生成处置任务
  → 派发至环保分局/园区/企业/第三方
  → 数据分析辅助决策
  → 现场处置/整改反馈
  → 结果归档 → AI核实
```

**流程二：人工发现异常**
```
园区/企业人员发现异常
  → 手动创建处置任务
  → 派发至环保分局/园区/企业/第三方
  → 数据分析辅助决策
  → 现场处置/整改反馈
  → 结果归档
```

### 3.2 数据采集与整合

数据资源中心采集与整合将为用户提供一个公共和完整的信息库。数据采集与整合主要是将收集的各项环境管理业务的基础数据进行数据加工处理、数据仓库分析和数据对比、匹配、校核，以解决同一数据来自不同业务的数据冗余和不一致问题。通过数据整合、校验，从源数据库中对业务基本数据按一定的主题、一定的规则进行抽取、清洗、转换，并加载到主数据库中，为数据展现提供支持服务。

#### 3.2.1 数据采集方式

根据环境数据的来源和格式，按照不同来源的更新机制，将数据集成进入数据资源中心库，数据集成方式有如下几种：

**（1）ETL数据导入包**

对于环保部统一下发软件或某些历史数据，通过开发ETL数据导入包，并通过相应的ETL管理工具，实现定期自动入库或手工入口，这其中还可能需要人工的干预。这也是主要的数据集成方式。

**实现方式：** 提供Excel/CSV导入模板，用户按模板填写数据后上传，系统解析并入库。

```
支持的数据源：
├── Excel模板导入（提供标准模板，用户填写后上传）
├── CSV文件导入
├── 历史数据库迁移（从环保部下发软件导出后导入）
└── API接口对接（与其他业务系统对接）
```

**（2）定期直接入库**

对于在线监测数据，由于数据的采集部门、方式和格式相对固定，不需要制定中间格式数据标准，直接由系统自动进行数据的检查和审核，以数据复制等方式将数据自动转入中心数据库。这种情况适合业务数据库不在数据资源中心库中，但数据资源中心库中相应部分的结构和业务数据库完全一致的情况。

**实现方式：** 通过MQTT/Modbus TCP协议直接接收在线监测设备数据，经协议解析后写入数据库。

**（3）数据录入**

对于某些数据，没有业务系统的支撑，需要通过特定的数据录入表单来进行数据采集入库。

**实现方式：** 提供Web表单录入界面，支持单条录入和批量导入。

**实现方式：** 提供XML交换规范，异地系统按规范生成XML文件，系统解析并入库。
